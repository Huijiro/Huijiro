### whoami

Engineer at [Agentuity](https://agentuity.com), the full-stack platform for AI
agents. Most of my work lives in the seams: the CLI, dev mode, the build
pipeline, the framework plumbing that makes spinning up an agent feel like one
command instead of a Friday afternoon.

Lately I've been leading our next major platform rewrite — retiring the old
runtime, swapping in a framework-agnostic build pipeline, wiring up first-class
support for the frameworks people actually want to use, and chasing every last
race condition out of dev mode.

Outside work I write Neovim plugins, PI extensions, and side-project games I
will absolutely not finish.

If a thing should be possible and isn't, I'll usually just write it.

### things I made that people actually use

- **[blink-cmp-supermaven](https://github.com/Huijiro/blink-cmp-supermaven)** — Supermaven source for [blink.cmp](https://github.com/Saghen/blink.cmp). The one repo of mine that strangers star.
- **[tmux-nav.nvim](https://github.com/Huijiro/tmux-nav.nvim)** — Minimal Neovim ↔ tmux pane navigation. Because every existing plugin had one feature too many.

### PI extensions

Building tools for [PI](https://github.com/earendil-works/pi-coding-agent), an extensible coding agent.

- **[click](https://github.com/Huijiro/click)** — SQLite-backed persistent memory with FTS5 search and auto-injection of relevant context.
- **[pi-lsp](https://github.com/Huijiro/pi-lsp)** — Inline LSP diagnostics on file reads, plus a standalone diagnostic tool.
- **[pi-fireworks](https://github.com/Huijiro/pi-fireworks)** — Fireworks.ai provider integration.
- **[nvim-pi](https://github.com/Huijiro/nvim-pi)** — PI inside Neovim.
